# CUJ-020: Security Audit

## User Goal
Audit system security and identify vulnerabilities.

## Trigger
- "Audit authentication security"
- "Security review"
- "Security audit"

## Workflow

### Step 1: Security Analysis
- Security Architect analyzes system
- Reviews authentication/authorization
- Checks for vulnerabilities

### Step 2: Threat Modeling
- Identifies threats
- Assesses risks
- Maps attack vectors

### Step 3: Audit Report
- Generates security audit report
- Lists vulnerabilities
- Provides remediation

### Step 4: Compliance Check
- Checks compliance requirements
- Validates standards
- Documents findings

## Agents Used
- Security Architect

## Skills Used
- Security scanning tools

## Expected Outputs
- Security audit report
- Vulnerability list
- Remediation plan
- Compliance checklist

## Success Criteria
- ✅ Vulnerabilities identified
- ✅ Risks assessed
- ✅ Remediation provided
- ✅ Compliance validated

## Example Prompts
```
"Audit authentication security"
"Security review of the API"
"Check for security vulnerabilities"
```
