# CUJ-003: Cross-Platform Setup

## User Goal

Sync LLM-RULES configuration across Claude Code and Cursor.

## Trigger

- "Sync to Cursor"
- "Set up cross-platform"

## Workflow

**Execution Mode**: `skill-only`

### Step 1: Identify Target Platform

- User specifies target: Cursor
- Check if target platform config exists

### Step 2: Context Bridge

- context-bridge skill exports current state
- Syncs agent configurations
- Syncs rule selections
- Syncs workflow state

### Step 3: Platform-Specific Setup

- For Cursor: Copy `.cursor/` configuration
- Validate platform-specific files

### Step 4: Verification

- Test in target platform
- Verify agents work
- Check configuration sync

## Agents Used

- None (skill-based)

## Skills Used

- `context-bridge` - Cross-platform sync

## Expected Outputs

- Synced configuration files
- Platform-specific setup complete
- Cross-platform compatibility verified

## Success Criteria

| Criterion                      | Measurement                       | Target                                                       |
| ------------------------------ | --------------------------------- | ------------------------------------------------------------ |
| Configuration synced correctly | File comparison between platforms | `.claude/` and target platform configs match                 |
| Target platform configured     | Platform-specific files exist     | `.cursor/` directory present with valid configs              |
| Agents work in target platform | Test agent invocation in target   | Agent responds to commands in Cursor                         |
| Context preserved              | Context comparison pre/post sync  | Workflow state, rule selections, and agent configs identical |

## Example Prompts

```
"Sync this context to Cursor"
"Set up Cursor IDE configuration"
```
