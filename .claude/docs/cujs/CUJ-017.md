# CUJ-017: Module Documentation

## User Goal

Generate claude.md file for a module.

## Trigger

- "Document the auth module"
- "Generate claude.md for [module]"
- "Create module documentation"

## Workflow

**Execution Mode**: `skill-only`

> **Note**: Skill-only CUJs execute directly without planning. If invoked via workflow, planning (Step 0) and rating (Step 0.1) occur first. Direct skill invocation skips planning and proceeds directly to skill execution.

### Step 1: Module Analysis

- Technical Writer analyzes module
- Extracts context from code
- Identifies patterns

### Step 2: claude.md Generation

- claude-md-generator creates file
- Follows template structure
- Includes module-specific info

### Step 3: Validation

- Validates claude.md structure
- Ensures completeness
- Checks consistency

## Agents Used

- Technical Writer

## Skills Used

- `claude-md-generator` - claude.md creation

## Expected Outputs

- claude.md file
- Module documentation
- Usage examples

## Success Criteria

| Criterion           | Measurement                        | Target                                                                 |
| ------------------- | ---------------------------------- | ---------------------------------------------------------------------- |
| claude.md generated | File exists in module directory    | File path: `<module_path>/claude.md`                                   |
| Structure correct   | Schema validation against template | Matches `.claude/templates/claude-md-template.md` structure            |
| Content complete    | Section count validation           | All required sections present (Purpose, Patterns, Dependencies, Usage) |
| Examples included   | Example code block count           | At least 2 usage examples with code snippets                           |

## Example Prompts

```
"Generate claude.md for src/modules/auth"
"Document the user management module"
"Create module documentation"
```
