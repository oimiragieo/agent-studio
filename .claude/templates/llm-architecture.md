### LLM Architecture Blueprint

- **Use Case**: task, users, constraints
- **Models**: primary/fallback, versions, budgets
- **Routing**: when to switch models/tools/skills
- **Context**: retrieval strategy, memory limits, redaction
- **Safety**: guardrails, PII handling, jailbreak mitigations
- **Evaluation**: metrics, datasets, success criteria
- **Ops**: logging, observability, rate limits, error handling
