# Codebase Walkthrough Prompt Template

## Purpose

Comprehensive codebase understanding and exploration.

## Template

```
I need to understand the codebase structure and architecture. Please:

1. **High-Level Overview**:
   - Identify the main technologies and frameworks used
   - Map the overall project structure
   - Identify key directories and their purposes
   - Understand the entry points and main flows

2. **Architecture Analysis**:
   - Identify architectural patterns (MVC, microservices, etc.)
   - Map component relationships and dependencies
   - Identify data flow patterns
   - Understand state management approach

3. **Key Components**:
   - List the main modules/components
   - Explain their responsibilities
   - Identify public APIs and interfaces
   - Map component interactions

4. **Dependencies**:
   - List external dependencies
   - Identify critical internal dependencies
   - Understand dependency relationships

5. **Configuration**:
   - Identify configuration files
   - Understand environment setup
   - Map configuration to functionality

Please provide a structured walkthrough that I can use to understand the codebase quickly.
```

## Usage

- Initial codebase exploration
- Onboarding new developers
- Architecture documentation
- Refactoring planning
